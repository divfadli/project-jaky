udah melampaui skala “Vendor IT” biasa → lebih mirip One Stop Service Agency.
======================

skema yang tadi kita bahas:
40–50% → Operasional (gaji tim, tools, internet, listrik, software)
20–30% → Profit Bersih Perusahaan
10–20% → Bonus Tim (disesuaikan dengan divisi/project)
5–10% → Dana Cadangan/Emergency

========================================

Alur Kerja Keuangan (Flow Per Project)
Request Masuk → Klien order (misalnya: Website eCommerce Rp50jt).
Estimasi Anggaran → Tentukan biaya (gaji tim, server, domain, dll.).
Kontrak & DP → Klien bayar DP (30–50%).
Eksekusi Project
-Catat semua pengeluaran (gaji, tools, transport)
-Simpan bukti transaksi (nota/invoice vendor)
Monitoring → Mingguan review apakah masih on-budget.
Project Selesai → Klien bayar pelunasan.
Distribusi Keuangan → Sesuai persentase tadi (operasional, profit, bonus, dana cadangan).
Laporan Akhir → Profit/Loss dicatat per project & per divisi.

===========================================

Contoh Breakdown Keuangan (Project Rp100.000.000)

Operasional (SDM, tools, listrik, internet, software, sewa) → Rp45.000.000
Profit bersih perusahaan → Rp25.000.000
Bonus tim (dibagi ke Project Manager, Dev, Designer, QA) → Rp15.000.000
Dana cadangan (emergency, bug fixing, revisi tambahan) → Rp10.000.000
Pajak/legal → Rp5.000.000

===================================================

-Gunakan kode project → setiap order dapat kode unik (misal: PROJ-WEB-2025-001).
-Pisahkan rekening bank khusus bisnis dari rekening pribadi.
-Buat laporan keuangan per divisi + gabungan supaya tahu divisi mana paling profit.
-Tetapkan SLA (Service Level Agreement) → supaya jelas kapan biaya tambahan dikenakan (misal revisi >3x ada biaya extra).
